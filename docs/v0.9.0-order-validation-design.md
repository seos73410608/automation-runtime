# v0.9.0 Order Validation

## Goal

Validate canonical order input data and generate validation reports.

## Input

jobKey
eventType
eventTime
userId
orderId
productId
amount
paymentMethod
traceId
correlationId

## Validation Rules

1. userId NOT NULL
2. orderId NOT NULL
3. productId NOT NULL
4. amount > 0
5. paymentMethod IN (CARD, BANK_TRANSFER, VIRTUAL_ACCOUNT)

## Output

validation_report.xlsx

## Metrics

total_rows
success_rows
failed_rows

## Runtime Flow

SOURCE
→ RULE
→ HISTORY
